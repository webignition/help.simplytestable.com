What This Error Means

This error is telling you that the validator reached the end of a <{{%0}}> element and found that this element did not contain what is expected.

This can happen if you inadvertently include an empty element which should contain child elements or if the markup within the element is not valid.

How To Fix It

For lists

Ensure that all lists contain list items. A list containing no list items is not a valid list.

Use: 

<{{%0}}>
  <li>Foo</li>
</{{%0}}>

instead of: 

<{{%0}}>
</{{%0}}>

Or remove the list entirely.

For the head of your document

  • ensure the required <title> is present

  • check for self-closing elements that should not be

  • check for incorrectly-encoded ampsersands

  • check for script or style elements lacking the type attribute

For noscript elements

Ensure that there are no errors in the markup contained in <noscript> elements.

Use:

<noscript>
  <p>
    Please enable JavaScript to view the <a href="http://disqus.com/?ref_noscript">comments.</a>
  </p>
</noscript>

<noscript>
  <p><img src="http://pixel.quantserve.com/pixel/example.gif" alt="" /></p>
</noscript>

instead of:

<noscript>
  Please enable JavaScript to view the <a href="http://disqus.com/?ref_noscript">comments.</a>
</noscript>

<noscript>
  <img src="http://pixel.quantserve.com/pixel/example.gif" alt="" />
</noscript>

For forms

Ensure that all form inputs are correctly within any required containing elements.

In your full HTML validation report, check for occurrences of the error document type does not allow element input here missing one of ….

Use:

<form action="/search/" method="get">
  <div>
    <input name="q" value="">
    <input type="submit" value="Search">
  </div>
</form>

instead of:

<form action="/search/" method="get">
  <input name="q" value="">
  <input type="submit" value="Search">
</form>

For table rows

A table row must contain cells to be considered valid.

Ensure all table rows contain a least one cell.

Use:

<table>
…
<tr>
  <td>Foo</td>
</tr>
…
</table>

instead of:

<table>
…
<tr>
</tr>
…
</table>

For description or definition lists

Ensure that all description or definition lists contain definitions. A list containing no definitions is not a valid list.

Use: 

<dl>
  <dt>Foo</dt>
  <dd>Bar</dd>
…
</dl>

instead of: 

<dl>
</dl>

Or remove the list entirely.

In general

There is no straightforward common fix for this type of error as it depends on the element that raised the error and the context in which this element appears in your markup.

Refer to the line and column number in the specific error you encountered to get some context on the cause of this error.

Check that you haven't included an element and omitted required child elements, such as including a list that lacks list items.

Check that the markup within the relevant element is valid. You may find that the relevant element contains additional elements that are not correctly nested or are not included within required containing elements.
