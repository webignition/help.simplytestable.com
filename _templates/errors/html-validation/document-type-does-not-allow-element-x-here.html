Most Likely Cause and Fix

You have placed a td element directly inside a table element without placing it inside a tr element.

Make sure all tables are correctly-formed.

Use 

<table>
  <tr>
    <td>Table data</td>
  </tr>
</table>

instead of 

<table>
  <td>Table data</td>
</table>

You have placed a style element outside the head of your markup.

Use 

<html>
  <head>
    <style>
    </style>
  </head>
  <body>
  </body>
</html>

instead of 

<html>
  <head>
  </head>
  <body>
    <style>
    </style>
  </body>
</html>

You have placed a <script> element after your closing </body> tag or after your closing </html> tag.

Use 

    <script></script>
  </body>
</html>

instead of 

  </body>
  <script></script>
</html>

and instead of 

</html>
<script>
</script>

Either:

Your HTML4 document contains an incorrectly closed single-line <meta> or <link> sibling in the <head> of your markup.

Use 

<meta name="keywords" content="documentation">
<link rel="copyright" href="#copyright">

instead of 

<meta name="keywords" content="documentation" />
<link rel="copyright" href="#copyright" />

Or:

You have JavaScript in a script element that contains the string <{{%0}}> which has not been marked as character data (CDATA).

Markup-like strings within JavaScript that is not marked as CDATA may be interpretted not as JavaScript but as markup. You do not want this.

Use 

<script type="text/javascript">
//<![CDATA[
var string = "<{{%0}}>";
//]]>
</script>

instead of 

<script type="text/javascript">
var string = "<{{%0}}>";
</script>

Either:

You have a <li> element preceded by a non-list element.

To add markup around a list item, place the markup inside the <li> element and not around it.

Use 

<ul>
  <li><a href="http://example.com">Example</a></li?>
</ul>

instead of 

<ul>
  <a href="http://example.com"><li>Example</li?></a>
</ul>

What This Error Means

HTML is often comprised of elements nested inside other elements, such as the placing of an anchor within a paragraph: 

<p>
    Hello <a href="http://example.com">World!</a>
</p>

There are rules regarding which elements can be nested inside other elements and when this is allowed to happen.

Most notably, block-level elements, such as <div>, are not allowed inside inline elements such as <span>.

This error relates to the when and is telling you that the <{{%0}}> element on the relevant line number is not allowed at that point.

How To Fix It

For table cells

Make sure all tables are correctly-formed.

Use 

<table>
  <tr>
    <td>Table data</td>
  </tr>
</table>

instead of 

<table>
  <td>Table data</td>
</table>

For style elements

Make sure all style elements are defined within the head of your markup and not elsewhere.

Use 

<html>
  <head>
    <style>
    </style>
  </head>
  <body>
  </body>
</html>

instead of 

<html>
  <head>
  </head>
  <body>
    <style>
    </style>
  </body>
</html>

For script elements

Place your <script> elements in the <head> of your markup or immediately before the closing </body> tag.

Use 

<html>
  <head>
    <script></script>
  </head>
  <body>
  </body>
</html>

or 

    <script></script>
  </body>
</html>

instead of anything else.

For strings in JavaScript

If the line and column number in the specific error message you encountered points to a section of JavaScript, you have a character data problem.

Markup-like strings within JavaScript that is not marked as CDATA may be interpretted not as JavaScript but as markup. You do not want this.

Use 

<script type="text/javascript">
//<![CDATA[
var string = "<{{%0}}>";
//]]>
</script>

instead of 

<script type="text/javascript">
var string = "<{{%0}}>";
</script>

In general

The specific error message you encountered will refer to a line and column number. This will guide you to the exact point in your markup to which the error relates.

Find the <{{%0}}> element in question and check to see what element it is inside of.

Check that you haven't failed to close a preceding sibling.

Check that the relevant <{{%0}}> is actually used in a place that it is allowed to be used.

Many elements have restrictions on the types of elements allowed inside them. For example, a list element can only have children that are list item elements.
